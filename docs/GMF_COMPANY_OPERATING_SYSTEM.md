# GetMeFound Company Operating System

Status: active source of truth
Owner: Mike Egidio
Purpose: the simple operating model for running GMF as autonomously as possible.

## Company Focus

GetMeFound helps local businesses stay visible and trusted as Google Search changes.

The company should not be positioned as a broad AI agency. GMF sells a narrow visibility system:

1. Get Found Refresh - one-time cleanup and visibility reset.
2. Stay Found - monthly monitoring and light upkeep.
3. Review Engine - email review requests for current and future customers.
4. Review Voice - add-on that drafts replies in the client's voice.
5. Call Protection - future add-on for answering calls when Google/customers call.

Reach, outbound prospecting, and social selling belong in a separate company/brand.

## Why This Exists Now

Google is moving more search activity into AI Mode, AI Overviews, follow-up answers, local agents, and richer source previews. Local businesses need their public footprint to be clean, current, trusted, and easy for Google/AI systems to understand.

GMF's customer-safe explanation:

Google is changing how people find businesses. We keep your business easier to find, trust, and choose.

## Offer Ladder

### Get Found Refresh

One-time service. Target price: $149.

Purpose: clean up the client's Google-facing footprint and give them a simple before/after.

Core work:

- Google Business Profile check
- category, service, hours, website, and description review
- review link capture
- local/AI-search readability check
- one-page findings and next-step report

### Stay Found

Monthly service. Target price: $49/mo.

Purpose: keep the profile and visibility signals from going stale.

Core work:

- monthly visibility check
- profile/review drift check
- simple monthly note: what changed, what needs attention, what is next
- client hub status

### Review Engine

Monthly service. Target price: $149/mo.

Purpose: send review requests to completed customers through GMF-owned email workflows.

Core work:

- customer upload/manual import
- POS-ready event path when available
- email review requests
- private feedback capture
- suppression/do-not-contact handling
- monthly review summary

### Review Voice

Monthly add-on. Target price: +$49/mo.

Purpose: draft review replies in the client's voice so they do not have to start from a blank page.

Safety rule: start in draft/approval mode. Do not auto-post until the client has proven trust, risk rules are stable, and Mike approves the automation level.

### Call Protection

Future add-on. Contact for pricing.

Purpose: answer or route calls when Google/customers call and the business cannot pick up.

Not a current default build. Keep it visible as future demand, not a blocker.

## How The Company Runs

Mike is the owner. Manager runs the operating system.

Manager assigns work, watches blockers, and only brings Mike decisions that need judgment, approval, money, or client risk.

Specialist agents own narrow jobs. They do not all read everything. Each agent gets the company focus, the client profile, and the workflow it owns.

Auditor checks quality and safety before work is marked done.

Coach keeps the knowledge base, sales language, and agent training clean.

## Where Mike Oversees

Mike should oversee from three places:

- `/mike-mc` - owner dashboard and active work.
- `/mike-mc/workflows` - workflow library and agent handoffs.
- `/mike-mc/ops` - company docs, services, and operating knowledge.

Mike should not need to inspect raw logs unless Manager or Auditor flags a problem.

## Autonomy Rules

- Agents do as much as they safely can before asking Mike.
- Manager asks Mike only for approvals, client-risk decisions, pricing, tool spend, credentials, or unclear judgment calls.
- Client emails drafted by agents require Manager review before Mike approval.
- Live sends, public profile edits, billing, and security-sensitive changes require proof.
- No HighLevel AI feature may be enabled without Mike's explicit approval.
- Supabase/app pages hold live status. Obsidian/docs hold knowledge and SOPs.

## Proof Before Done

No workflow is done until proof exists:

- profile/access proof for Refresh and Stay Found
- send proof for Review Engine
- draft/approval proof for Review Voice
- health/safety proof for weekly audits
- client-visible summary for monthly retention

## Weekly Rhythm

Daily:

- Manager scans stuck work and urgent approvals.
- Systems checks broken pipes only when health signals show risk.

Weekly:

- Profile Manager checks visibility drift.
- Reviews Manager checks review send/feedback status.
- Auditor checks stuck workflows and safety gates.

Monthly:

- Client Success/Coach produces the client recap.
- Manager shows Mike clients at risk, renewals, upgrades, and blockers.
